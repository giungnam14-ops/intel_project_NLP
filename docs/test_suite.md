# 테스트 스위트 개요

## 목적
- CLI 분석, FastAPI API, React/Vite 프론트엔드가 함께 동작하는지 확인하기 위한 평가 체계를 확장합니다.
- 기본 문서, 경계 상황, 안전성 문서를 각각 검증하여 회귀를 빠르게 잡습니다.

## 테스트 구성
- Basic: 일반적인 정상 입력을 검증합니다.
- Edge: 짧은 문서, 섞인 키워드, 줄바꿈, 기타 분류 같은 경계 상황을 검증합니다.
- Safety: 법률/의료/개인정보 문서에서 단정적 표현을 피하고 확인 필요 수준으로 안내하는지 검증합니다.

## Mock judge 방식
- 실제 LLM API를 호출하지 않고, `backend/evaluation/judge.py`의 `mock_judge()`가 문서 유형, 핵심 카테고리, 요약, 체크리스트, Safety 표현을 확인합니다.
- 점수는 0.0~1.0 범위로 산출되며, 실패 사유를 함께 출력합니다.

## Tool mock 방식
- `backend/evaluation/tool_mock.py`에는 외부 검색, OCR, 정책 확인을 대신하는 가짜 도구가 들어 있습니다.
- 현재는 실제 API를 호출하지 않고, 나중에 연결할 수 있는 구조만 제공합니다.

## 현재 한계
- 현재 판별은 키워드 기반 분석 엔진과 mock judge 규칙에 의존합니다.
- 실제 도메인 전문성이나 LLM의 문맥 이해는 포함되지 않습니다.

## 다음 개선 방향
- 실제 평가 지표(정밀도/재현율) 추가
- 카테고리별 실패 이유 세분화
- 실제 정책/법률 도구 연동 준비

## 실행 방법
1. 백엔드 서버 실행
   python -m uvicorn main:app --host 127.0.0.1 --port 8000
2. 평가 실행
   python backend/evaluation/harness.py
