# 문제 후보 [A]

## 누구의 문제인가 (대상 사용자)
- 약관, 동의서, 계약서, 공지문, 안내문처럼 긴 생활 문서를 빠르게 확인해야 하는 일반 사용자
- 문서가 길거나 어려워서 핵심 내용만 먼저 알고 싶은 사용자
- 중요한 조건이나 주의 문장을 놓치기 쉬운 사용자

## 어떤 상황에서 겪는가
- 앱이나 웹사이트 가입 과정에서 긴 약관과 개인정보 동의서를 확인해야 할 때
- 무료체험, 구독 서비스, 쇼핑몰 결제 과정에서 자동결제나 환불 조건을 확인해야 할 때
- 학교, 회사, 병원, 기관 등에서 받은 공지문이나 안내문에서 날짜, 장소, 준비물, 제출 기한을 확인해야 할 때
- 계약서나 동의서에서 금액, 책임, 해지 조건처럼 중요한 내용을 빠르게 파악해야 할 때

## 반복적으로 겪는 페인 포인트 (관찰/인터뷰 근거)
- 문서가 길고 표현이 어려워 처음부터 끝까지 읽기 부담스럽다.
- 중요한 조건이 문서 중간에 숨어 있어 사용자가 놓치기 쉽다.
- 요약을 봐도 실제로 무엇을 조심해야 하는지 바로 알기 어렵다.
- 결제, 환불, 개인정보 제공, 해지 조건, 제출 기한처럼 놓치면 불편이나 손해가 생길 수 있는 항목을 직접 찾아야 한다.
- 사용자는 문서를 읽기보다 빠르게 넘기거나, 필요한 부분만 대충 검색해서 확인하는 경우가 많다.

## 지금은 어떻게 (불편하게) 해결하고 있나
- 문서를 처음부터 끝까지 직접 읽어본다.
- Ctrl+F로 ‘환불’, ‘결제’, ‘개인정보’, ‘기한’ 같은 단어를 직접 검색한다.
- 이해가 안 되는 문장은 인터넷이나 ChatGPT에 따로 물어본다.
- 시간이 없을 때는 자세히 읽지 않고 동의하거나 넘어간다.
- 공지문이나 안내문은 캡처해두었다가 나중에 다시 확인하지만, 중요한 날짜나 준비물을 놓치는 경우가 있다.

## 이 문제가 중요한 이유
- 생활 문서에는 사용자가 놓치면 실제 불편이나 손해로 이어질 수 있는 중요한 조건이 포함되어 있다.
- 자동결제, 환불 제한, 개인정보 제공, 해지 조건, 제출 기한 같은 내용은 사용자가 반드시 확인해야 하지만 문서 안에서 찾기 어렵다.
- 기존 요약 방식은 내용을 줄여줄 수는 있지만, 사용자가 바로 확인해야 할 위험 문장이나 핵심 조건을 직관적으로 보여주지는 못한다.
- 따라서 긴 문서를 다시 읽지 않아도 핵심 조건과 주의 문장을 5초 안에 확인할 수 있는 서비스가 필요하다.
