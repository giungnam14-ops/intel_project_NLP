# MVP 성공 기준

## 한 문장 성공 기준

사용자가 긴 생활 문서를 입력했을 때, 서비스가 문서 유형을 분류하고 핵심 정보와 주의 문장을 카드 및 체크리스트 형태로 보여주면 MVP는 성공이다.

## 성공 기준 상세

## 1. 문서 입력이 가능해야 한다

사용자는 약관, 공지문, 논문·보고서 등 긴 문서를 텍스트로 입력할 수 있어야 한다.

## 2. 문서 유형을 분류할 수 있어야 한다

서비스는 입력된 문서가 어떤 유형인지 분류할 수 있어야 한다.

MVP에서 우선 다루는 문서 유형은 다음과 같다.

- 약관 / 동의서
- 공지문 / 안내문
- 논문 / 보고서

## 3. 문서 유형별 핵심 정보를 추출할 수 있어야 한다

문서 유형에 따라 서로 다른 핵심 정보를 추출해야 한다.

약관 / 동의서:
- 자동결제
- 환불 제한
- 개인정보 제공
- 해지 조건

공지문 / 안내문:
- 날짜
- 장소
- 준비물
- 제출 기한
- 신청 방법

논문 / 보고서:
- 연구 목적
- 연구 방법
- 핵심 결과
- 한계점

## 4. 결과를 카드 형태로 보여줘야 한다

분석 결과는 긴 요약문이 아니라 사용자가 빠르게 확인할 수 있는 카드 형태로 제공되어야 한다.

각 카드에는 다음 정보가 포함된다.

- 카테고리
- 핵심 내용
- 쉬운 설명
- 중요도 또는 위험도
- 원문 문장

## 5. 체크리스트를 생성해야 한다

서비스는 사용자가 확인하거나 행동해야 할 항목을 체크리스트로 제공해야 한다.

예시:

- 자동결제 시작일 확인
- 환불 가능 여부 확인
- 제출 기한 확인
- 준비물 확인
- 연구 결과 확인

## 6. 사용자가 5초 안에 핵심을 파악할 수 있어야 한다

결과 화면은 복잡하지 않아야 한다.  
사용자가 긴 문서를 다시 읽지 않아도 중요한 내용을 바로 확인할 수 있어야 한다.

## 7. 시연이 가능해야 한다

발표 시 다음 3가지 문서 예시를 입력했을 때 각각 다른 결과가 나와야 한다.

- 약관 예시
- 공지문 예시
- 논문/보고서 예시

## MVP 성공 조건 요약

- 문서 텍스트 입력 가능
- 문서 유형 분류 가능
- 핵심 정보와 주의 문장 탐지 가능
- 카드형 결과 출력 가능
- 체크리스트 생성 가능
- 약관, 공지문, 논문 예시로 시연 가능

## MVP 실패 조건

다음 중 하나라도 해당하면 MVP 성공으로 보기 어렵다.

- 문서를 입력해도 결과가 나오지 않는다.
- 모든 문서에 같은 형태의 일반 요약만 출력된다.
- 문서 유형별로 다른 핵심 정보를 보여주지 못한다.
- 결과가 너무 길어 사용자가 다시 읽어야 한다.
- 체크리스트가 생성되지 않는다.
- 발표 시연에서 약관, 공지문, 논문 결과 차이가 보이지 않는다.
