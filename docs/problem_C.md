# 문제 후보 [C]

## 누구의 문제인가 (대상 사용자)
- 학교, 회사, 병원, 기관 등에서 받은 공지문이나 안내문을 확인해야 하는 사용자
- 긴 안내문에서 날짜, 장소, 준비물, 제출 서류, 신청 방법만 빠르게 확인하고 싶은 사용자
- 공지 내용을 읽고도 무엇을 해야 하는지 정리가 잘 안 되는 사용자

## 어떤 상황에서 겪는가
- 학교나 학원에서 받은 가정통신문, 행사 안내문, 제출 안내문을 확인할 때
- 회사나 기관에서 받은 교육, 행사, 신청 안내문을 확인할 때
- 병원, 관공서, 행사 안내문에서 날짜와 준비물을 확인해야 할 때
- 신청서 제출, 서류 준비, 장소 방문처럼 사용자가 직접 해야 할 일이 있는 문서를 읽을 때

## 반복적으로 겪는 페인 포인트 (관찰/인터뷰 근거)
- 공지문이 길어서 핵심 내용만 빠르게 찾기 어렵다.
- 날짜, 장소, 준비물, 제출 기한 같은 중요한 정보를 놓치기 쉽다.
- 안내문을 읽었는데도 실제로 무엇을 해야 하는지 바로 정리되지 않는다.
- 준비물이나 제출 서류를 뒤늦게 확인해 불편을 겪는 경우가 있다.
- 공지문을 캡처해두고도 나중에 다시 찾아 읽어야 해서 번거롭다.

## 지금은 어떻게 (불편하게) 해결하고 있나
- 공지문을 직접 읽고 날짜나 준비물을 따로 메모한다.
- 중요한 부분을 캡처하거나 형광펜처럼 표시해둔다.
- 제출 기한이나 장소를 캘린더에 따로 입력한다.
- 헷갈리는 내용은 주변 사람에게 물어본다.
- 나중에 다시 공지문을 열어 확인한다.

## 이 문제가 중요한 이유
- 공지문과 안내문에는 사용자가 실제로 해야 할 일이 포함되어 있다.
- 날짜, 준비물, 제출 서류, 장소 같은 정보를 놓치면 일정에 늦거나 준비물을 빠뜨리는 불편이 생길 수 있다.
- 단순 요약보다 사용자가 해야 할 일을 체크리스트로 정리해주는 방식이 더 실용적이다.
- 따라서 공지문과 안내문에서 핵심 정보와 해야 할 일을 자동으로 추출해주는 서비스가 필요하다.
